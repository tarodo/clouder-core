# Accessibility Checklist · CLOUDER iter-2a

> Minimal a11y baseline. Not a full WCAG 2.2 AA audit. Public-launch audit is a separate ticket.

## 1. Focus visible

All interactive elements (`<button>`, `UnstyledButton`, custom button-divs) must show a visible focus ring on `:focus-visible` of colour `var(--color-border-focus)` with 2px offset. Mantine 9 renders this by default for its primitives once the theme exposes the CSS variable. Custom components (`DestinationButton`, `ListItem`) add the same ring manually.

## 2. Icon-only buttons require `aria-label`

Every `<ActionIcon>` and any custom icon-only button has an explicit `aria-label`. Examples:

- Hotkey overlay trigger: `aria-label="Show keyboard shortcuts"`.
- TrackRow kebab: `aria-label={`Actions for ${track.title}`}`.
- Player play/pause: `aria-label={state === "playing" ? "Pause" : "Play"}`.

## 3. Custom components — ARIA contract

### DestinationButton
- `role="button"` (UnstyledButton already provides).
- `aria-keyshortcuts="<digit>"` when `hotkey` prop is set (`aria-keyshortcuts="1"`).
- `aria-pressed={state === "primary"}` when `primary` is the meaningful selected state.
- `aria-disabled={state === "disabled"}` (UnstyledButton's `disabled` prop already handles this).

### CategoryPill
- Use `<UnstyledButton>` (NOT `<Badge>`) so the pill is keyboard-activatable.
- `aria-pressed={selected || undefined}` when toggleable.

### ListItem
- Container has `role="listbox"` (or `role="menu"` for action menus).
- Items have `role="option"` (or `role="menuitem"`) with `aria-selected` reflecting state.
- Keyboard: ↑/↓ moves focus, Enter activates. Use Mantine's `useUncontrolled` + `useFocusReturn` patterns; do not roll a hand-coded loop.

### TrackRow
- The row is `<UnstyledButton>` with `aria-label={track.title}` (gives the row a name).
- Selection a11y is owned by the leading `<Checkbox>` — do NOT use `aria-pressed` on the row.
- Unavailable rows: `aria-disabled={track.unavailable}` + `tabIndex={-1}` so keyboard users skip them.
- Kebab `<ActionIcon>` uses `aria-label="Track actions"` and `event.stopPropagation()` to prevent firing the row click.

### PlayerCard
- Play/pause `<ActionIcon>` carries `aria-label={state === "playing" ? "Pause" : "Play"}`.
- Action affordances inside the subline (Retry, Open device picker) use `<Anchor component="button">` not bare `<a onClick>` so they are keyboard-activatable and correctly focusable.

### Hotkey overlay (Modal)
- Mantine `Modal` provides focus trap by default.
- On open: focus moves to first interactive child (the close button).
- On close: focus returns to the trigger (Mantine handles via `useFocusReturn` if `returnFocus` prop is `true`, which it is by default).

### BlockHeader
- Disabled `Finalize` button: wrap in `<Tooltip label="Triage all tracks first" disabled={unclassifiedCount === 0}>` so the disabled state has a discoverable explanation.

Tokens.css ramp passes WCAG AA at the relevant pairings (oklch sources rounded to perceptual lightness):

Document this baseline so future palette tweaks have a regression target.

## 5. Reduced motion

Disable vestibular-triggering effects (transform, parallax, large translateY) on `prefers-reduced-motion: reduce`. Keep non-vestibular feedback (opacity fades, instant colour swaps, focus rings) — these aid comprehension and don't trigger motion sensitivity.

Per-component rules:

- DestinationButton `just-tapped` `scale(0.97)` → drop transform, keep colour pulse.
- Multi-select bar slide-up → fade-in only, no translateY.
- Skeleton shimmer → static neutral background, no animation loop.

Avoid the universal-selector reset (`* { animation-duration: 0 !important }`) — it overrides the per-component decisions above. Instead apply the media query directly inside each component's CSS:

```css
.destination-button.just-tapped {
  transform: scale(0.97);
  transition: transform var(--motion-pulse) var(--ease-out);
}

@media (prefers-reduced-motion: reduce) {
  .destination-button.just-tapped { transform: none; }
}
```

## 6. Lang attribute

Per the i18n decision, iter-2a ships English copy: `<html lang="en" {...mantineHtmlProps}>`.

## 7. Live regions

Transient feedback must reach screen readers, not just sighted users:

- **Mantine `<Notifications>`** — wrap its outlet in `aria-live="polite"`. Mantine 9's `Notifications` component does NOT do this automatically; the consumer adds the attribute on the wrapper.
- **Multi-select bar** — when count changes (e.g. "3 tracks selected"), update an `aria-live="polite"` region inline rather than re-rendering silently.
- **Finalize / long-op statuses** — for the 3-stage copy escalation (B7, OPEN_QUESTIONS Q9), the loading text is in an `aria-live="polite"` region so SR users hear "Cold start, hang on…" → "This is taking longer than usual…".
- **Just-tapped feedback in Curate** — visual-only is OK because the auto-advance to the next track produces an aural state change anyway via track title. No additional live region needed.

## 8. Touch targets

iPhone Air (primary mobile, 420 CSS px wide) — minimum interactive area **44 × 44 px** per iOS HIG. Visual size of a control may be smaller, but its hit-target must reach the minimum via padding.

| Component | Visual height | Hit area | How |
|---|---|---|---|
| CategoryPill | 24 px | 44 px (compact) / 48 px (regular) | wrap in `<UnstyledButton>` with `padding: 10px 12px` |
| ActionIcon (kebab, transport) | 32–36 px | 44 px | `size="lg"` on mobile / `min-height: 44px` |
| TrackRow | 48 px (compact) / 64 px (regular) | meets minimum already | — |
| Tab (Tabs.Tab) | 36 px | 44 px | bottom `padding: 4px` extends hit area downward |
| DestinationButton (mobile lg) | 64 px | 64 px | already exceeds minimum |

Where padding is impractical, document the divergence (e.g. inline tag in dense table) and ensure neighbours don't make accidental selection likely.
