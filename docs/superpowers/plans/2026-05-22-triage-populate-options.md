# Triage Populate Options Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add two optional inputs to "create triage block" — an OLD-depth offset (in weeks) and a toggle that routes tracks with user-disliked labels into the NOT bucket at highest classification priority.

**Architecture:** Track classification happens once, at block creation, inside a single `INSERT … SELECT` in `TriageRepository.create_block`. We (1) persist the two inputs on the `triage_blocks` row for audit, (2) compute `old_cutoff = date_from − N weeks` in Python and substitute it for the hardcoded `date_from` in the OLD test, and (3) when the disliked toggle is on, conditionally prepend a `WHEN EXISTS (… disliked label …) THEN NOT` branch to the `CASE`. A pure Python mirror (`classify_bucket_type`) is kept in sync. Defaults (`0` / `false`) reproduce today's behavior exactly. The frontend adds a collapsed "Advanced" section to the create dialog.

**Tech Stack:** Python 3 + Aurora via RDS Data API (no psycopg at runtime), Pydantic v2 schemas, Alembic migrations, pytest. Frontend: React 19 + Mantine 9 + Zod + react-i18next + Vitest. OpenAPI generated by `scripts/generate_openapi.py`; frontend types by `openapi-typescript`.

```python
def classify_bucket_type(
    *,
    spotify_release_date: date | None,
    release_type: str | None,
    old_cutoff: date,
    is_disliked: bool = False,
) -> str:
    """Pure mirror of the SQL CASE in create_block.

    Ordering (first match wins):
        is_disliked          -> NOT   (highest priority)
        NULL date            -> UNCLASSIFIED
        date < old_cutoff    -> OLD
        compilation          -> NOT
        else                 -> NEW
    """
    if is_disliked:
        return BUCKET_TYPE_NOT
    if spotify_release_date is None:
        return BUCKET_TYPE_UNCLASSIFIED
    if spotify_release_date < old_cutoff:
        return BUCKET_TYPE_OLD
    if release_type == "compilation":
        return BUCKET_TYPE_NOT
    return BUCKET_TYPE_NEW
```

```python
            # 5. Classify and insert tracks (one INSERT FROM SELECT).
            #    Disliked-label branch is highest priority and only present
            #    when the toggle is on (built as a SQL string fragment so we
            #    avoid binding a no-op subquery when off).
            disliked_when = ""
            if include_disliked_labels:
                disliked_when = """
                        WHEN EXISTS (
                            SELECT 1
                            FROM clouder_albums a
                            JOIN clouder_user_label_prefs ulp
                              ON ulp.label_id = a.label_id
                            WHERE a.id = t.album_id
                              AND ulp.user_id = :user_id
                              AND ulp.status = 'disliked'
                        ) THEN :not_bucket_id
                """
            self._data_api.execute(
                f"""
                INSERT INTO triage_bucket_tracks
                    (triage_bucket_id, track_id, added_at)
                SELECT
                    CASE
                        {disliked_when}
                        WHEN t.spotify_release_date IS NULL
                            THEN :unclassified_bucket_id
                        WHEN t.spotify_release_date < :old_cutoff
                            THEN :old_bucket_id
                        WHEN t.release_type = 'compilation'
                            THEN :not_bucket_id
                        ELSE :new_bucket_id
                    END,
                    t.id,
                    :now
                FROM clouder_tracks t
                WHERE t.style_id = :style_id
                  AND t.publish_date BETWEEN :date_from AND :date_to
                  AND NOT EXISTS (
                    SELECT 1
                    FROM category_tracks ct
                    JOIN categories c ON ct.category_id = c.id
                    WHERE c.user_id = :user_id
                      AND c.style_id = :style_id
                      AND c.deleted_at IS NULL
                      AND ct.track_id = t.id
                  )
                """,
                {
                    "user_id": user_id,
                    "style_id": style_id,
                    "date_from": date_from,
                    "date_to": date_to,
                    "old_cutoff": old_cutoff,
                    "now": now,
                    "new_bucket_id": tech_bucket_id_by_type[BUCKET_TYPE_NEW],
                    "old_bucket_id": tech_bucket_id_by_type[BUCKET_TYPE_OLD],
```
